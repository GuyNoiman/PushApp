# Screen_Bible.md

# Purpose

This document defines the experience of every major screen inside PushApp.

Unlike the Information Architecture document, this document does not focus on navigation or technical architecture.

Instead, it explains:

- Why each screen exists.
- Which user need it serves.
- What emotional state it should create.
- Which use cases it supports.
- Which information deserves visual priority.
- Which interactions should be optimized.
- Which ideas belong to the future rather than the MVP.

This document should become the primary reference for every UX designer, Product Manager and AI working on PushApp.

---

# UX Philosophy

Every screen in PushApp exists for one reason only.

It should answer one primary question.

When a screen tries to solve several unrelated problems, it becomes confusing.

PushApp should therefore optimize for clarity rather than information density.

The application should gradually feel familiar rather than overwhelming.

Whenever a user opens a screen they should immediately understand:

• Why they are here.
• What they can do.
• What should happen next.

---

# Global UX Principles

Every screen should:

• Have one primary CTA.
• Minimize cognitive load.
• Prioritize real-life progress over in-app engagement.
• Make progress visible.
• Never shame the user.
• Celebrate achievements whenever appropriate.
• Reduce configuration whenever possible.
• Let Buddy feel alive throughout the experience.

---

# Navigation Philosophy

Current preferred bottom navigation:

Home

Journeys

Explore

Friends

Buddy

Additional areas:

Inbox

Settings

These remain accessible but should not occupy a permanent bottom-tab position.

The navigation should remain stable over time.

Users should rarely need to learn new navigation patterns.

# HOME

## Purpose

The Home screen is the decision engine of PushApp.

It answers one question:

> **What should I do right now?**

Nothing on this screen should distract from helping the user continue making progress today.

---

## Emotional Goal

The user should feel:

"I know exactly what deserves my attention."

Not:

"I'm overwhelmed by everything I need to do."

---

## Primary CTA

Complete the next Step.

---

## Secondary CTAs

Help a friend.

Open today's Journey.

Open overdue Step.

Claim rewards.

---

## Primary Information

Buddy Level

XP

XP remaining until next level

Coins

---

## Hero Section

Today's highest priority Step.

If nothing is scheduled today:

Suggest planning.

Suggest Explore.

Suggest helping an Ally.

---

## Today's Steps

Highest priority first.

Order:

1. Overdue
2. Due today
3. Recommended today

Each Step should clearly communicate urgency.

Examples:

Today

Tomorrow

Overdue

Last chance this week

---

## Weekly Progress

A separate section showing:

Current week's commitment.

Examples:

Run

2 / 3

Spanish

1 / 2

Meditation

5 / 7

The objective is helping users understand their remaining weekly commitments.

---

## Global Missions

Separate from Journeys.

Examples:

Help 3 friends.

Spin today's wheel.

Invite one friend.

Send a gift.

Complete 5 Steps.

These primarily reward Coins.

Not XP.

---

## Friends Needing Help

Highly visible.

Helping others is considered a desirable behaviour.

Users should quickly understand:

Who currently needs support?

Primary actions:

Chat

Poke

Send Gift

Encourage

---

## Reward Visibility

XP progress.

Coins.

Recent Achievement.

Recent Buddy unlock.

Progress should always feel visible.

---

## Future Ideas

Adaptive Home.

Buddy-generated priorities.

Dynamic widgets.

Personalized Home ordering.

> **What am I currently trying to become?**

---

## Emotional Goal

---

## Main Content

Future Journeys.

Completed Journeys.

## Journey Details

Opening a Journey reveals:

Rules.

Future Steps.

---

## Future Steps

Users should always be able to inspect the complete Journey.

They should understand where they are heading.

---

## Journey Rules

Examples:

Pause allowed.

Restart conditions.

Completion requirements.

Not Home.

**what they should do next to become closer to the person they choose to be.**

Every UX decision should reinforce that outcome.
