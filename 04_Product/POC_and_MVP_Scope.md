# POC / MVP Scope

Status: 🚧 MISSING DOCUMENT — to be written together (do not define independently)

---

This document will define the Proof of Concept and the MVP scope for PushApp.

It does not yet exist **by design**. Per the 2026-07-06 decisions (see `06_Decisions/Decision_Log.md`, D4), the POC/MVP scope must be defined jointly with the founder and should not be authored independently.

## What this document will eventually contain

- The single POC hypothesis and how it will be validated.
- The minimum feature set for the MVP.
- What is explicitly out of scope for the first version.
- The one metric / experiment that de-risks the next step.

## Known inputs so far (not yet a definition)

- Tentative POC direction (from founder, not finalized): user-created Journeys + an add-friends mechanism + a basic Support Circle (friends only) + basic notification adaptation + friend/progress updates.
- Principle: AI is part of the MVP but no core flow depends on it (Decision_Log D2).
- Positioning: young adults building meaningful habits and goals across life areas (Decision_Log D1).

Until this document is written, treat POC/MVP scope as **undefined**.
