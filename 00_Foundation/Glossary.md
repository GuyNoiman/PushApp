# Glossary

Status: Approved

---

# Purpose

This document defines the official terminology used throughout the PushApp Knowledge Repository.

Its purpose is to establish a shared language across product, strategy, research, development, design, and AI collaboration.

Every official term should have exactly one agreed meaning.

Whenever a new product concept becomes part of PushApp, it should be added to this glossary.

---

# Structure

Each term should follow the same structure:

## Definition

A concise description of what the term means.

## Purpose

Why this concept exists and the value it brings to PushApp.

## Notes

Important clarifications, limitations, or distinctions from similar concepts.

## Related Terms

Links or references to other glossary terms.

---

# Repository Rule

Whenever a new official concept is introduced during product discussions, ask:

**"Is this an official PushApp term?"**

If the answer is yes, it should be added to this glossary using the standard structure.

The glossary is a living document and should evolve together with the product.
